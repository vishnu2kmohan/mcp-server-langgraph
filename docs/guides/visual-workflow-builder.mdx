<Note>
**Visual Workflow Builder** enables non-developers to design agent workflows visually and export production-ready Python code.
</Note>

## Overview

The Visual Workflow Builder provides:

- **Drag-and-Drop Canvas** - React Flow-based visual workflow design
- **5 Node Types** - Tool, LLM, Conditional, Approval, Custom
- **Code Export** - Generate production-ready LangGraph Python code
- **Code Import** - Round-trip: import existing code back to visual
- **Templates** - Pre-built workflow patterns

### Design Your First Workflow

<Steps>
  <Step title="Add Nodes">
    Drag nodes from the palette onto the canvas:
    - **Tool Node** 🔧 - Execute functions (web search, database query)
    - **LLM Node** 🧠 - Call language models (Claude, GPT, Gemini)
    - **Conditional Node** 🔀 - Route based on state conditions
    - **Approval Node** ✋ - Human-in-the-loop checkpoints
    - **Custom Node** ⚙️ - Custom Python functions
  </Step>

  <Step title="Connect Nodes">
    Click and drag from a node's output handle to another node's input:
    - Regular edges: Direct flow
    - Conditional edges: Add conditions (e.g., `state['score'] > 0.8`)
  </Step>

  <Step title="Configure Nodes">
    Click a node to configure:
    - **Label**: Display name
    - **Config**: Node-specific settings (model, tool name, etc.)
  </Step>

  <Step title="Generate Code">
    Click **Generate Code** to see production-ready Python in the Monaco editor.
  </Step>

  <Step title="Export">
    - **Copy to Clipboard**: Quick copy
    - **Download**: Save as `.py` file
    - **Save Workflow**: Save JSON for later editing
  </Step>
</Steps>

## Node Types

### Tool Node 🔧

Execute tools and functions:

```yaml
Config:
  tool: "web_search"      # Tool name
  args: {}                # Default arguments
```

Generated code:
```python
def node_search(state):
    """Execute Search - tool: web_search."""
    result = call_tool("web_search", state)
    state["result"] = result
    return state
```

### LLM Node 🧠

Call language models:

```yaml
Config:
  model: "claude-sonnet-4-5"     # Model identifier
  temperature: 0.7                   # Sampling temperature
  system_prompt: "..."               # Optional system prompt
```

Generated code:
```python
def node_analyze(state):
    """Execute Analyze - LLM: claude-sonnet-4-5."""
    from litellm import completion

    response = completion(
        model="claude-sonnet-4-5",
        messages=[{"role": "user", "content": state["query"]}],
        temperature=0.7
    )
    state["llm_response"] = response.choices[0].message.content
    return state
```

### Conditional Node 🔀

Route based on state:

```yaml
Config:
  conditions:
    - condition: "state['score'] > 0.8"
      target: "approve"
    - condition: "state['score'] <= 0.8"
      target: "retry"
```

Generated code:
```python
def route_decision(state):
    """Route based on conditions."""
    if state['score'] > 0.8:
        return "approve"
    elif state['score'] <= 0.8:
        return "retry"
    return "default"

graph.add_conditional_edges("decision", route_decision, {
    "approve": "approve_node",
    "retry": "retry_node",
    "default": END
})
```

### Approval Node ✋

Human-in-the-loop checkpoints:

```yaml
Config:
  risk_level: "high"      # high, medium, low
  timeout: 3600           # Seconds to wait for approval
  approvers: ["admin"]    # Required approvers
```

Generated code:
```python
def node_approval(state):
    """Human approval checkpoint."""
    # Pauses workflow for human approval
    # Configured for high risk level
    return interrupt(state, approvers=["admin"], timeout=3600)
```

    "nodes": [
      {
        "id": "search",

        "label": "Search",
        "config": {"tool": "web_search"},
        "position": {"x": 100, "y": 100}
      },
      {
        "id": "summarize",
        "type": "llm",
        "label": "Summarize",
        "config": {"model": "claude-sonnet-4-5"},
        "position": {"x": 300, "y": 100}
      }
    ],
    "edges": [
      {"from_node": "search", "to_node": "summarize", "label": ""}

## Generated Code Example

**Input Workflow**:
- Node 1: Web Search (tool)
- Node 2: Summarize (LLM)
- Edge: search → summarize

**Output Python**:

```python
"""
Research agent

Auto-generated from Visual Workflow Builder.
"""

from typing import TypedDict
from langgraph.graph import StateGraph


class ResearchAgentState(TypedDict):
    """State for research_agent workflow."""
    query: str
    result: str


def node_search(state):
    """Execute Search - tool: web_search."""
    result = call_tool("web_search", state)
    state["result"] = result
    return state


def node_summarize(state):
    """Execute Summarize - LLM: claude-sonnet-4-5."""
    from litellm import completion

    response = completion(
        model="claude-sonnet-4-5",
        messages=[{"role": "user", "content": state["query"]}]
    )
    state["llm_response"] = response.choices[0].message.content
    return state


def create_research_agent():
    """Create research_agent workflow."""
    graph = StateGraph(ResearchAgentState)

    graph.add_node("search", node_search)
    graph.add_node("summarize", node_summarize)
    graph.add_edge("search", "summarize")
    graph.set_entry_point("search")

    return graph.compile()


# Usage:
# agent = create_research_agent()
# result = agent.invoke({"query": "AI trends 2025"})
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Frontend not loading">
    ```bash
    # Check nginx serving static files
    curl http://localhost:13000

    # Check API connectivity from frontend
    curl http://localhost:9001/api/builder/node-types
    ```
  </Accordion>

  <Accordion title="Code generation fails">
    - Ensure workflow has at least one node
    - Verify `entry_point` matches a valid node ID
    - Check edges reference existing nodes
  </Accordion>

  <Accordion title="Import not working">
    - Code must use `StateGraph` from `langgraph.graph`
    - Must have `add_node` and `add_edge` calls
    - Must have `set_entry_point` call
  </Accordion>
</AccordionGroup>
